## 3、核心矛盾:homo数据量远大于cross,这正是主动学习存在的原因

用户指出的问题——homo多、cross少——是这个项目从round2起就在正面应对的核心矛盾,现状是:

| | 规模 | 说明 |
|---|---:|---|
| homo(同分子自缩合) | **~219k-220k对**,champion MAE 1.503 | 全库220,859个醛,几乎一醛一对(donor==acceptor),已全部DFT标注 |
| cross候选池(candidates_v3) | 200万对(400万有向) | 仅占理论组合空间C(220859,2)≈244亿的**约0.0082%** |
| cross已标注(round1-7) | **16,241对/32,456行** | 仅为homo已标注规模的**~7.4%** |
| cross剩余可用候选(scaffold-clean、未标注) | **126.3万对** | 07-17盘点确认,足够支撑round8及以后多轮 |

也就是说,**候选池本身不是瓶颈**(结构多样性已验证与全库随机抽样一致),**真正的瓶颈是
DFT标注吞吐量**——这正是round2起就采用不确定度主动学习(bootstrap ensemble std排序,
校准性已验证Spearman 0.258、十分位单调)而不是均匀抽样的原因:用有限的DFT算力标注
"模型最不确定"的那部分数据,单位标注成本换来的泛化提升更大。
