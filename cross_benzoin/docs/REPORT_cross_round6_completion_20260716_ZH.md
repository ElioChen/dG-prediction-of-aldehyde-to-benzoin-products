有两点值得注意：

1. **单模型XGB champion的冻结集MAE从第5轮到第6轮变差了**（2.983→3.132），这是整个项目里这个指标第一次出现回退。与此同时CV MAE却在改善（2.397→2.276），所以这不是模型整体性能下降——而是随着训练数据分布变化，这29行的冻结集恰好对这个特定架构"变得不利"。考虑到n=29，这在一定程度上是冻结集这一侧本身的抽样方差，而非真实能力下降，但这正是为什么现在应该把ensemble（而不是单模型XGB）当作当前最佳、已验证的产出物，而不是把这次回退当成需要解释掉的数据故障。
2. **MLP+XGB ensemble持续变好**（2.633→2.582），在目前尝试过的每一个规模上都保持最佳。

## 3. 第5轮的GNN-blend胜利未能复现

上一份报告强调了一个stacking结果：在第5轮规模下，ensemble和GNN的固定50/50 blend以bootstrap显著性击败了单独的ensemble（`analyze_gnn_stack_uncertainty.py`，P(blend更优)=0.987，n=29）。在第6轮训练表上重新训练GNN，并重跑**完全相同**的分析脚本，得到了不同的答案：

| | 第5轮 | 第6轮 |
|---|---:|---:|
| 仅ensemble MAE | 2.777 | **2.547** |
| 仅GNN MAE | 2.624 | 2.820 |
| 50/50 blend MAE | 2.520 | 2.564 |
| P(blend优于ensemble) | 0.987 | **0.456** |
| P(仅GNN优于ensemble) | 0.767 | 0.096 |

（注意：分析脚本打印出的第5轮"仅ensemble"MAE为2.777，与第2节里ensemble自己报告的冻结集MAE 2.633略有差异——分析脚本是在GNN也有有效预测的那29行交集上重新计算预测的，这与ensemble自己独立做冻结留出集评估的口径有极细微差别；这里重要的是对比的方向和显著性，不是小数点后第三位。）

因为两次跑的冻结测试集是完全相同的29行，这是一次真实的反转，不是测试集侧的噪声：ensemble在这些行上的误差下降了（2.777→2.547），而GNN的误差上升了（2.624→2.820）。GNN是用**与第5轮完全相同的架构和超参数**重新训练的（同源预训练三编码器，针对约扩大46%的训练集没有做任何调参尝试）——最可能的解释是在新规模下欠调参，而不是GNN在这个任务上存在根本性能天花板。`train_cross_gnn.py`自己内部的blend搜索（连续权重扫描，不同于不确定性脚本测试的固定0.5 blend）在第6轮找到的最优GNN权重只有0.15（metadata记录：`best_blend_w_gnn: 0.15`，`best_blend_mae: 2.531`，相比仅ensemble的2.547），这是一个小得多、几乎可以忽略的优势，比不确定性脚本测试的50/50 blend温和得多——与"GNN在这个规模下能贡献的东西相对第5轮变少了"这一判断是一致的。

**解读**：第5轮"blend获胜"的结论应视为**暂定、已被推翻**，而不是可以继续building的既定事实。目前唯一最佳、已验证的产出物是单独的ensemble（`data/cross_benzoin/cross_round6/train_ensemble_6rounds_slim120_v1/models/cross_ensemble_model.joblib`，冻结集MAE 2.582，R² 0.897）。GNN stacking是否值得继续投入现在是一个开放问题——需要在第6轮规模上做一次认真的调参（学习率、训练轮数、模型容量）之后才能下结论，而不是仅仅在更多行上重跑同一套配方。

## 4. 尚未决定的事项

- **模型晋升**：第6轮的MLP+XGB ensemble是目前最佳、已验证的模型，但尚未被晋升为`score_round_active_learning.py`给未来主动学习轮次打分所使用的正式模型（目前仍在用单模型XGB champion这条线）。
- **GNN调参**：在信任或放弃stacking这个想法之前，值得认真做一次超参数搜索——第5轮的结果当时看起来是真实的胜利，结果并不是；但仅凭一次在新规模上的重训练，也不足以彻底否定它。
- **第7轮**：第6轮用掉4,000对之后，screen10k候选池里还剩3,677/7,677对未使用——第7轮可以直接从剩余部分抽取，零新增xTB/g-xTB计算，和第6轮做法一样。
- **同源+交叉合并，第三次测量**：此前的会话发现合并收益随交叉自身数据增长而萎缩（4,120行时相对MAE改善-1.9% → 17,270行时-1.0%）。第6轮（25,176行）是观察这个趋势是趋于平坦、继续萎缩、还是反转的自然下一个数据点。

## 结论

第6轮干净收尾，没有数据完整性问题（0个DFT错误，计算成功率维持在一贯的约94%区间）。MLP+XGB ensemble依然是最佳、已验证的交叉模型，并且再次改善（冻结集MAE 2.633→2.582）。不过本轮最重要的发现其实是一个否定性结果：第5轮报告的GNN-stacking胜利，在第6轮规模下用完全相同的分析重新检验后发生了反转，不应被当作已经定论的科学结果——这提醒我们，在n=29的冻结留出集上，即使是bootstrap显著性检验，单一轮次的显著结果也需要在至少再一个规模上做复现检查，才能真正信任。
