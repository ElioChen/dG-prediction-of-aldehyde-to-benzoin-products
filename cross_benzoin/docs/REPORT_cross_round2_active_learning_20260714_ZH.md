## 本轮任务背景

第一轮（`cross_pilot_v1`，详见
`REPORT_cross_pilot_dft_sp_validation_20260714_EN.md`）建立了第一个真实的
交叉苯偶姻 Δ-学习模型：598 行/299 个无序配对，在6类反应物类别组合上做均匀
多样性采样，CV MAE 2.106，对比 g-xTB 基线 3.44。当时留下的开放问题是：
**模型的主要瓶颈是模拟分子数太少——如果有针对性地增加数据，是否真的能改善？**

第二轮用一个真正的主动学习闭环来回答这个问题，而不只是再做一批随机/多样性
采样：

1. 构建了一个4200行的多样性采样候选池（`cross_round2`，与第一轮不同的
   donor/acceptor 配对，类别重新配平）。完成 xTB+g-xTB 特征计算（job
   24614015）和产物侧 BDE 计算（job 24620376）——两者都很干净（成功率分别
   为99.5%和93.5%，失败模式与第一轮相同、均为良性失败）。
2. 训练了一个**40成员的bootstrap集成模型**（对第一轮299个配对做有放回重
   采样，xgb配置与已上线模型相同），对全部4181行有效的第二轮数据打分。按
   **预测不确定性（集成内标准差）**对候选排序——这是真正由认知不确定性
   驱动的主动学习：问的是"在重采样扰动下，第一轮模型对自己最不一致的地方
   在哪里"，而不只是"哪些分子在化学结构上看起来不一样"。
3. 按不确定性选出**前900/2098个配对**（43%）送去做DFT-SP标注（1794条有
   方向的行，两个取向都保留，与第一轮的约定一致）。
4. 运行DFT-SP（job 24620959，r2SCAN-3c，18任务数组，genoa分区）：在实际
   处理的集合中**1756/1756行有效，零错误**（1794行中有38行因为其中一侧
   醛的DFT-SP能量落在全库已知的~0.8%超时缺口里，被提前排除，而不是被
   静默算错）。
5. 把第一轮+第二轮合并成一张150特征的训练表（2354行/1179个配对），重新
   训练 `train_cross_delta.py`。

## 结果1 —— 不确定性排序确实挑出了真正难的化学体系，不是噪声

被选中配对的类别分布 vs 第二轮全池：

| 反应类别        | 被选中 | 池中总数 | 选中比例 |
|-----------------|-------:|---------:|---------:|
| carbo-hetero    | 252 | 399 | 63% |
| hetero-hetero   | 216 | 400 | 54% |
| carbo-carbo     | 204 | 399 | 51% |
| aliph-carbo     |  88 | 300 | 29% |
| aliph-hetero    |  86 | 300 | 29% |
| aliph-aliph     |  54 | 300 | 18% |

模型的bootstrap分歧集中在杂原子/碳环芳香类配对上，而不是均匀分布的。解读：
第一轮的299个配对已经比较充分地约束了脂肪族配对的行为（脂肪醛结构更简单、
方差更低），而杂原子/碳环芳香化学空间方差更大、覆盖不足。这正是主动学习
应该产出的那种有针对性的结果，也是一种比第一轮"纯多样性配平"更有依据的
优先级排序。

在正式提交整批DFT-SP之前做了独立的合理性检验：一个3行冒烟测试显示，同一
被选中配对的两个取向（donor/acceptor互换）给出了**方向一致的DFT修正量**
（相对g-xTB分别为+8.5和+9.5 kcal/mol）——模型标记为"不确定、值得标注"的
那些大修正量是真实的、系统性的化学效应，不是几何噪声。

## 结果2 —— 第二轮选中的数据集在两个基线上都明显更难

|                        | 第一轮 (n=598) | 第二轮 (n=1756) |
|------------------------|---------------:|-----------------:|
| `dG_orca_kcal` 均值    | 5.06 | 6.42 |
| `dG_orca_kcal` 标准差  | 5.03 | 9.61 |
| `dG_orca_kcal` 最大值  | 24.29 | 52.58 |
| g-xTB 基线 MAE         | 3.44 | 6.24 |
| Δ-模型 MAE（合并模型CV中）| 2.01 | 3.53 |

这正是主动学习筛选按设计应有的效果：第二轮本质上就是富集了g-xTB（以及第
一轮模型）处理得最差的那些配对。**合并模型的整体CV MAE（3.147）看起来比
第一轮单独训练的2.106更差——这是数据难度分布发生了变化，而不是模型能力
退化。** 只在第一轮的数据行上评估合并后的模型，MAE仍是2.01（与第一轮自身
样本内的数字持平甚至略好），证实加入更难的第二轮数据并未造成模型性能下降。

## 结果3 —— 在难样本上，模型相对g-xTB的优势反而变大了

|                              | 仅第一轮 | 合并（第一轮+第二轮） |
|------------------------------|---------:|----------------------:|
| CV Δ-模型 MAE                | 2.106 | 3.147 |
| CV g-xTB 基线 MAE             | 3.445 | 5.534 |
| **相对g-xTB的MAE改善量**       | **+1.339** | **+2.387** |
| CV R²                        | 0.708 | 0.749 |
| 冻结留出集 n_test             | 14 | 29 |
| 冻结留出集 Δ-模型 MAE         | 2.638 | 3.612 |
| 冻结留出集 g-xTB MAE          | 3.078 | 5.315 |
| 冻结留出集 R²                 | 0.697 | 0.831 |

这里有两点比原始MAE数字更重要：(1) 模型相对g-xTB基线的**绝对改善幅度几乎
翻倍**（+1.34 → +2.39 kcal/mol），因为随着化学体系变难，g-xTB退化得比
Δ-模型快得多——这正是Δ-学习方法的核心价值主张，而现在这一点在"第一轮之外
的新化学体系"上得到了验证，而不仅仅是样本内表现。(2) 冻结的、分子不重叠
的留出集（基于candidates_v3的真实train/test划分，是这条流水线里唯一完全
无泄漏的评估指标）规模翻倍（14→29个测试行），其R²从0.697提升到0.831——
这是一个更可信的数字，且仍然显示Δ-模型相对g-xTB有巨大优势。

## 结果4 —— 特征分组消融：与第一轮模式一致

| 特征分组         | 特征数 | MAE | RMSE | R² |
|------------------|-------:|----:|-----:|---:|
| 仅2D特征         | 48  | 4.391 | 5.874 | 0.543 |
| 仅醛类特征       | 54  | 3.994 | 5.393 | 0.615 |
| 仅产物特征       | 54  | 3.699 | 5.023 | 0.666 |
| donor+acceptor   | 102 | 4.012 | 5.415 | 0.612 |
| 全部原始分组     | 140 | 3.126 | 4.325 | 0.752 |
| 全部+交互项      | 150 | 3.147 | 4.356 | 0.749 |

产物侧描述符单独一组（`product_only`）仍然是单一最强的特征分组，合并所有
原始分组仍带来最大的提升——与第一轮结论一致。`interaction_*`（donor-acceptor
互补性交互项）这一组再次没有带来可测量的额外提升（3.126 vs 3.147，处于CV
噪声范围内）——与第一轮的空结果相同，现在在约4倍样本量下再次得到确认。除非
出现更廉价的计算变体，否则未来默认训练流程中可以考虑去掉这一组。

## 本轮尚未解决的已知缺口

- **数据划分一致性问题**：冻结留出集评估这一步丢弃了2354行中的788行（原因
  是donor/acceptor醛在`candidates_v3`的分子不重叠划分表中找不到，或落在
  不同的划分里）——与此前push会话中记录的根因相同（自定义配对生成脚本没有
  像`candidates_v3`自身的400万配对列表那样尊重"每个分子只能属于一个划分"
  的约束）。第三轮应直接从`candidates_v3`采样以消除这部分损失。
- **~38行醛缓存缺口**：未解决，与全库同源苯偶姻DFT-SP活动中已知的~0.8%
  超时长尾问题相同。在当前规模下不值得专门回填，未来轮次中会（以递减规模）
  再次出现。
- **ENSEMBLE72打包缺口**：`predict_dG()`的推理路径仍然没有调用真正产出本
  报告讨论的任何Δ-模型（第一轮冠军模型、本轮合并模型，或另一个独立的同源
  全库ENSEMBLE72冠军模型）所用的现代特征流水线——上一次会话中已标记，
  刻意未仓促处理，目前仍然开放。

## 结论

以不确定性驱动（而非单纯多样性采样）的主动学习正确识别出了第一轮模型最薄
弱的化学体系（杂原子/碳环芳香类配对），针对这部分化学体系补充真实DFT标注
后，模型相对g-xTB基线的优势在一个更难、更多样的数据群体上几乎翻倍，同时
冻结留出集规模更大、更可信（R² 0.697 → 0.831）。表面上的MAE"退步"
（2.106 → 3.147）完全可以由所选数据群体变难来解释，而不是模型质量下降——
这一点已通过"合并后模型在仅第一轮数据上评估、MAE基本不变（~2.0）"得到
证实。这验证了继续推进第三轮主动学习闭环的合理性。
