# 从反应物 / 产物 / 二者结合预测 ΔG:三种框架定义的化学问题

> 概念性思考记录 · 2026-06-20
> 背景:benzoin 缩合 `2 R-CHO → R-CO-CH(OH)-R`,目标量 `dG = G(product) − 2·G(reactant)`(r2SCAN-3c // xTB)。

---

## 0. 前提:ΔG 是"边属性",不是"节点属性"

预测的 `dG = G(product) − 2·G(reactant)` 本质上是一个**状态函数的差值**——它属于"反应"这条边,而不属于反应物或产物任何一个分子(节点)。这一点决定了后面所有的不对称:

> 从反应物预测、从产物预测,本质上都是**用一条边的一个端点去回归这条边的标签**。它们之所以能 work,靠的是冗余;它们之所以不等价,靠的是冗余在不同方向上会断裂。

在 benzoin **自偶联**里,反应物 → 产物是一个**双射**(一个 R-CHO 唯一决定 R-CO-CH(OH)-R)。所以在信息论意义上,反应物、产物、二者结合三者**携带的信息完全相同**。但作为**科学问题**和**学习问题**,它们定义了三个截然不同的东西。

---

## 1. 从反应物预测 ΔG —— "底物 scope / 正向反应性"问题

**化学空间**:可枚举的醛库(22 万,本质是 PubChem 上可买到的、线性规模 N 的分子集合)。**密集采样、供给驱动(push)**。

**科学语义**:实验化学家手上的问题——"我货架上这个醛,是不是好底物?"模型学的是**取代基电子/位阻效应如何调制羰基的热力学**(EWG/EDG、共轭、杂芳、邻位)。以反应物为中心的描述符(Mulliken/ADCH 电荷、PA、Fukui)天然服务于这个框架。

**为什么反应物侧特别 learnable**:ΔG 的变化主要发生在反应中心(C=O π 键断裂 + 新 C–C + 生成 α-羟基酮)。两端 R 相同,大量贡献相消,ΔG 实际上是"被差值放大的反应中心局部变化";而这个局部变化对 R 的依赖,恰好是**反应物羰基能被 R 稳定多少**——一个可以直接从反应物读出的性质。所以反应物→ΔG 的映射相对平滑、近似可加。

**内在局限**:它**默认了自偶联的产物拓扑**,是整张交叉反应矩阵的**对角线**。对 cross-benzoin(两个不同醛)在定义上失效——你需要一个"对"。

---

## 2. 从产物预测 ΔG —— "逆合成 / 目标可及性"问题

**化学空间**:benzoin / α-羟基酮空间。自偶联子集规模 N(对角线),但**完整非对称产物空间 ~ N²/2 ≈ 2×10¹⁰**——比反应物空间大十个数量级。**组合爆炸、需求驱动(pull)**。

**科学语义**:目标导向合成——"我想要这个 α-羟基酮(药物中间体、手性砌块、天然产物骨架),它热力学上有多容易从醛前体拼出来?"模型必须**内化那个断键**:α-羟基酮的 `C(=O)–C(OH)` 就是 retron,产物模型在隐式地"定位 retron + 评估两个碎片的热化学"——本质是**隐式逆合成**。

**独有能力**:产物**显式编码配对的不对称性**。`R¹-CO-CH(OH)-R²` 里 R¹≠R² 的信息单个反应物无法表达。所以**产物框架天然外延到 cross-benzoin / 非对称产物**。`featurize_product.py --emit-aldehydes`(产物特征化 + 反推组成醛)就是把这个断键工程化了。

**深层 caveat(开放科学问题)**:若产物模型**只在自偶联数据上训练**,它只见过产物空间的**对角线**,对**非对角(非对称产物)外推未经验证**。核心问题:**碎片可加性成立吗?**

$$\Delta G(AB) \overset{?}{\approx} \tfrac12\big(\Delta G(AA)+\Delta G(BB)\big) + \text{交叉项}$$

检验这个可加性、刻画交叉项大小,**本身就是一个可发表的科学结论**。

---

## 3. 二者结合 —— "反应空间 / 网络热力学"问题

**化学空间**:不再是节点,而是**双部反应图的边**——(反应物集合, 产物)对。对该转化最忠实的表示。给模型两个端点 = 给它真实的热力学循环端点。"结合"真正解锁三件单端点做不到的事:

**(a) 选择性(chemo-/regio-)只活在这里。** 对两个醛 A、B,存在 ΔG(AA)、ΔG(BB)、ΔG(AB) 三条边——**选择性是它们的相对排序**。单端点一次只看一条边;只有共享端点的联合表示能自洽地同时给出三者,预测"homo vs cross 竞争"谁胜出。

**(b) 节点势 vs 边回归,以及 Hess 一致性(最深的一点)。**
若不直接回归 ΔG,而是学**每分子的 G(一个"节点势")**再做差,则**整张反应网络上任意循环自动求和为零(Hess 定律按构造满足)**。直接从反应物或产物回归 ΔG 的模型,**在 cross/homo 构成的循环上可以违反 Hess 定律**,给出热力学不自洽预测。

> 支持"结合/端点"作为**正确热力学模型**的最强论据:学节点势 → 免费得到全网络一致的热力学场;直接回归 ΔG → 省事但可能违反守恒。

**(c) Δ-learning 在这里才有原理。** xTB 端点 + ML 修正:对 `G_product` 和 `G_reactant` 各自用廉价物理打底,再学残差。物理基线已把"差值结构"编码进去,修正项可以是反应物侧、产物侧或两侧——"结合"不是简单拼特征,而是**让 Δ-learning 的循环自洽**。

---

## 4. 一个统一的对偶:同一个数,两种读法

benzoin 缩合**可逆**(retro-benzoin 真实存在),同一个 ΔG 在两个框架下有对偶解读:

| ΔG | 反应物框架(scope, push) | 产物框架(retrosynthesis, pull) |
|---|---|---|
| 强放热 | **好底物**,产物稳定 | 目标**易及且抗逆解**,好合成目标 |
| 吸热 | 差底物 | benzoin **不稳定、会回退**,难目标 |

"供给驱动地发现好底物"与"需求驱动地评估目标可及性"是**同一物理的两面**。

---

## 5. 对项目的落点(分层建议)

呼应"build BOTH, 一个 joint aromatic model"的 hierarchy 方向:

1. **反应物-only 2D surrogate** → 大规模底物筛选(对 22 万库做 push)。便宜、平滑、可加性强。
2. **产物-only / 碎片模型** → 逆合成可及性 + 进军 cross-benzoin 的**非对角空间**;但必须先用少量交叉数据**验证可加性假设**,否则是外推。
3. **结合/端点(学 per-molecule G + Δ-learning)** → 作为**物理自洽的主干**:支持选择性、保证 Hess 闭合、给 cross 提供原理性基线。当作前两者的"物理锚",而非又一个黑箱回归。

**一句话总结三个框架定义的科学问题:**
- **反应物 = 我能用什么(reactivity / scope)**
- **产物 = 我想做什么(retrosynthesis / accessibility)**
- **结合 = 这个转化的热力学全貌(reaction-network thermodynamics,唯一能谈选择性与守恒的层次)**
