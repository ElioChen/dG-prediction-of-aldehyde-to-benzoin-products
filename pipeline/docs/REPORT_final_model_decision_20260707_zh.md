# 最终模型决策：MORDREDSLIM271_BDEGXTB —— 2026-07-07 后续分析批次综合总结

## 结论

**MORDREDSLIM271_BDEGXTB 仍是生产冠军模型**（275 特征：72 个 QM + 199 个 SHAP 精简 mordred +
4 个 g-xTB BDE/BDFE；MLP + XGB_d8 + XGB_d10 + 分位数不确定性集成；测试集 MAE **1.503**，
RMSE 2.257，R² 0.875；bundle 路径
`pipeline/models/gxtb_dft_correction_MORDREDSLIM271_BDEGXTB_20260706.joblib`）。为了
(a) 对该模型做健全性检查与解读，(b) 尝试在架构层面（包括用 GNN）超越它，共进行了四项后续
分析。没有一项真正超越它；以下各节总结每项分析的发现，以及为何该决策维持不变。

## 1. SHAP 成本审计——保留模型，简化未来的特征获取

在 4000 行测试子样本上做 SHAP 分析（XGB_d8）显示，4 个 g-xTB BDE/BDFE 特征的用处并不对等：
`ald_bde_gxtb_kcal`（排名 4/275）和 `prod_bde_gxtb_kcal`（排名 6/275）——两者都便宜（单次
SP/opt）——的 SHAP 重要性求和是 `prod_bdfe_gxtb_kcal`（排名15）和 `ald_bdfe_gxtb_kcal`
（排名38）的 **3.7 倍**，而后两者需要对每个片段做昂贵的完整 Hessian `--ohess`。**行动建议：
未来对新分子做前瞻性筛选时，只计算 BDE，跳过 BDFE**——现有 bundle 两者都保留（全 22 万库已经
算过，没有理由为了删掉 2 个已算好的列而重新训练），但这一发现排除了未来在 BDFE 族（热/熵）
描述符上继续投入的必要性。完整方法论见
`REPORT_shap_mordredslim271_bdegxtb_20260707.md` / `_zh.md` 以及
[METHODS_BDE_gxtb_20260707_zh.md](METHODS_BDE_gxtb_20260707_zh.md)。

## 2. 深度误差/噪声带分析——剩余 1.503 kcal/mol 误差从何而来

32.3% 的测试误差（21,910 个分子中的 7,075 个）真实地高于 baseline_72 重新打乱种子研究建立的
噪声地板（1.571±0.013 kcal/mol，3-sigma 截断值 1.610）——其余约 68% 的误差落在纯 DFT 标签
噪声可以解释的范围内，无法靠更好的模型修复。在真实误差长尾中：**磺酰基富集 11.2 倍，含磷
9.5 倍，亚胺 3.6 倍，酰胺 3.4 倍（新发现的驱动因素），酯 1.8 倍，硝基 1.5 倍**。结构空间
（Morgan 指纹）聚类发现 4 个相似的、以卤素为主导的困难簇（平均误差 2.8-3.0）+ 1 个以酰胺为
主导的簇（3.36，最差）+ 1 个无主导标签的异质簇——与此前 SHAP 归因空间"4 种异质失败模式"的
发现一致。**含义：如果要继续追求精度提升，应针对这些具体的官能团失败模式（例如专门的磺酰基/
酰胺特征，或有针对性的 DFT 重新标注），而不是泛泛的模型/架构改动**——见新增的噪声带直方图
`126_noise_band_histogram_champion275_20260707.png`。

## 3. 芳香族子集深入分析——历史采样偏差已解决，脂肪族现在是更难的部分

全库芳香族占比（66.3%）与测试集占比（66.3%，差距 0.03 个百分点）一致——旧有的 MaxMin
主动选择对芳香族的欠采样偏差在当前全库训练群体中**已不存在**（该问题特定于早期的主动选择
阶段，如今 DFT-SP 已覆盖近乎全部 clean_v4 库，该问题已不成立）。按范围划分的精度：**芳香族
MAE 1.327/R²0.875，脂肪族 MAE 1.870/R²0.842**——脂肪族现在是更难的部分，不确定度路由也正确
反映了这一点（route-to-DFT 比例 26.5% 对 9.7%）。芳香族内部，化学型的影响很小（简单碳环
1.268 最好，嗪/唑类含氮 1.391，呋喃 1.375，噻吩 1.275——都很接近）。**含义：如果优先安排
进一步的数据采集，脂肪族分子是价值更高的目标，而非芳香族。**

## 4. GNN 追平尝试——确认架构不是关键杠杆

`gnn_dual_qm_champion275.py`（任务 24478591）给双编码器 GINE 架构（完整架构扫描中表现最好
的 GNN 架构，`REPORT_homo_v6_gnn_architectures_20260629.md`）在读出层喂入与冠军模型完全
相同的 275 个特征，在同一个匹配约 21.8 万分子群体上使用相同的 70:20:10 种子 42 划分协议——
测试的是更丰富的特征（而非不同的架构）能否让 GNN 追平或超越 1.503。**结果：测试集 MAE
1.552**（RMSE 2.269，R²0.874，54.4 万参数）——大幅缩小了此前的差距（dual_qm+56特征版本是
1.646，如今用 275 特征降到 1.552），但仍落后表格模型冠军 0.049。结合此前完整架构扫描
（不论用哪种卷积算子，所有 2D 卷积算子都在 2.58-2.68 见顶；只有注入 QM 特征才能打破这个
平台期）以及更早的 3D-GNN 结果（SchNet 2.18，输给同样数据下的 MLP 1.83），这一结论已在三个
独立测试中得到稳健验证：**即使给 GNN 与表格模型完全相同的信息，它依然无法像梯度提升树那样
充分利用这些信息。** 对这个任务而言，架构替换不是关键杠杆，信息（特征）才是。

## 5. 架构提升尝试：GNN + 表格模型堆叠集成

既然不同架构和特征对齐都未能让 GNN 单独获胜，架构层面剩下的杠杆是**组合**这两类模型家族，
而非用一个取代另一个——GNN（消息传递）和 GBDT（轴对齐分裂）是足够不同的函数类别，它们的
误差可能只是部分相关，若如此，即使两者都不能单独获胜，组合也可能超越两者。
`gnn_dual_qm_champion275_ensemble.py`（任务 24482531）用完全相同的协议重新训练了同一个
双编码器 GNN，同时额外保存了带分子 ID 的逐分子测试集预测，再与表格模型冠军已保存的测试集预测
（`test_predictions_MORDREDSLIM271_BDEGXTB_20260706.csv`）在权重扫描
`w_gnn ∈ {0, 0.05, ..., 1.0}` 上做混合。

**中途发现并修复了一个方法论 bug（任务 24482531）：第一次尝试的混合对比结果是无效的。**
两个脚本都用 `np.random.default_rng(42).permutation(N)` 做 70:20:10 划分，但各自独立推导
自己的群体规模 `N`（GNN 脚本与表格模型最终脚本的缺失值处理略有不同）——即使种子相同，对不同
N 做排列本质上是两个不相关的随机序列，所以各自选出的"最后 10%"测试集几乎没有交集（任务
24482531 的运行中断言失败：`unexpectedly small overlap`，退出码 1，正是证实了这一点）。
**修复方法**（`gnn_ensemble_reeval_matched.py`）：不再依赖 GNN 自身独立划分恰好对齐，而是
**复用已训练好的 GNN 权重（24482531 保存），直接在表格模型测试集的确切分子 ID 上重新评估**。
这一修复还暴露出一个更严重、必须过滤掉的风险：在表格模型 21,858 个测试集 ID 中，**有 15,258
个（70%）其实落在了 GNN 自己的训练集里**——这正是两个独立划分对同一群体取交集时按概率预期
应有的约 70% 重合率——必须先剔除这些，否则 GNN 对它们的"记忆"（而非泛化）预测会虚高其表面
精度。

**在干净、无泄漏的交集上的结果（n=6,613 个分子，任务 24489591）：**

| 配置 | MAE（n=6,613 子集） |
|---|---|
| 仅表格模型 | 1.476 |
| 仅 GNN | 1.557 |
| **最佳混合（w_gnn=0.40，即 60% 表格模型 + 40% GNN）** | **1.425** |

相对仅表格模型在该子集上的差值：**-0.051**，明显超出已确立的噪声带（0.02-0.03 kcal/mol）
——混合确实带来了真实提升，与"GNN 和 GBDT 的误差只是部分相关"的假设一致。**需要说明的
局限**：这个结果是在约 6,600 个分子的子集（无泄漏交集）上测得的，而非完整的 21,910 行官方
测试集，且该子集上仅表格模型的 MAE（1.476）与完整测试集上官方报告的 1.503 并不相同——所以
这个 -0.051 的差值只在该子集内部比较时严格成立，尚不能直接断言完整生产模型会从 1.503 降到
约 1.45。**采纳前需要先确认的下一步**：从头开始，用一个共享、对齐的 70:20:10 划分（两个
模型采用同一套基于 ID 的折分配）重新训练两个模型，这样就能在完整测试集上做混合，而不需要
这种子集化处理，并确认该提升在全量规模下依然成立。

## 总体建议

目前仍将 MORDREDSLIM271_BDEGXTB 作为唯一的生产模型，但**堆叠集成的结果足够有希望（-0.051
MAE，约为噪声带的 2 倍，且是在真实但部分重合的交集上得到的），值得先做上述对齐划分的重新
训练**，再决定是否在生产打分中采用混合预测器（代价是推理时也要额外运行 GNN）。如果希望在
集成结果之外进一步提升精度，还有两条有证据支持的路径：(a) 针对第 2 节中官能团失败模式
（磺酰基/酰胺/亚胺/磷）做定向工作，(b) 在未来任何标注工作中优先考虑脂肪族分子（第 3 节）。
不建议继续投入独立 GNN 替代方案（第 4 节）或 BDFE 族描述符（第 1 节）。
