# 一周进展总结（2026-06-29 ~ 2026-07-06）

覆盖范围：DFT-opt 公平重跑 → ORCA 并行修复 → g-xTB 溶剂化中试 → descriptor 搜索 → BDE/BDFE 攻关 → 共享环境事故与修复 → 训练规范化。本周最大的产出是 **g-xTB→DFT 校正模型的连续三次提升**（1.61 → 1.525 → 1.503，最终版已跑完整 MLP+3×XGB 生产管线，非快速检验）。

## 1. 基础设施 / 工作流可靠性改进

### 1.1 修正了一个可能误导全部结论的 bug：ORCA 并行 = MPI 假象
- **问题**：`concl3` 图（06-22）里 "DFT-opt 0/13 收敛" 曾被当作"DFT 优化本身失败"的证据，但排查发现所用的 `nprocs 24` 并行提交其实连 `mpirun` 都没找到 —— **是 ORCA 启动失败，不是 SCF 不收敛**。
- **根因**：ORCA 6.1.1 需要 OpenMPI 4.1.x ABI（`libmpi.so.40`），集群自带的 OpenMPI 只有 4.1.5（不兼容）和 5.0.x（错误 ABI）。
- **修复**：自行在 node-local 编译了 **OpenMPI 4.1.6**（`software/build_ompi416_local.sh`，本地盘编译 ~15min vs GPFS 上 50min+），并定位到真正的隐藏坑——`OMPI_MCA_rmaps_base_oversubscribe` / `hwloc_base_binding_policy` 两个环境变量，否则即使库对了 ORCA 仍会 "error termination in Startup"。
- **后续验证**：用**串行 nprocs=1** 重跑那 36 个"g-xTB-hard"分子的 DFT-opt（job 24300065），得到干净、无 MPI 干扰的真实收敛数据，避免了此前可能写进最终报告的错误结论。
- **沉淀为可复用配方**：单点计算继续用"分子级并行 + 串行 ORCA"（生产线一直如此），几何优化才需要这套并行修复方案。

### 1.2 SLURM 运维踩坑与规范化
- `scontrol update ArrayTaskThrottle` 事后调整并发度**不可靠**——同样的命令，有时正常工作，有时会把整个 array 直接 CANCELLED（发生过 2 次意外取消，好在都在任务刚起步、无输出损失时发现）。**新规矩**：并发度一律在 `sbatch --array=...%N` 提交时就设好，事后修改必须立刻用 `sacct` 核实没有被误杀。
- `clean_node_orphans.sh`（每个 job 启动时的自清理脚本）过去存在**两个隐藏 bug**（scontrol 退出码判断错误 + 从路径里把用户名 "schen3" 里的 "3" 当成了 jobid），两个 bug 互相抵消导致清理"什么都没删"，直到 nodespecific 目录堆到 270 万 inode 撞上配额上限才暴露。本周彻底修复并验证（171 个确认死亡目录安全删除，存活 job 的 scratch 全部保留）。

### 1.3 共享 Python 环境（`envs/gnn`）两次崩溃 → 彻底修复
- 本周 `envs/gnn` 因为一次被中断的批量重装，**158/160 个包（包括 torch 本身）**处于"看似能 import、实际残缺"的状态，一度让 g-xTB BDFE 全库计算出现 ~30% 静默失败（SLURM 还显示 COMPLETED，因为提交脚本末尾无条件 `echo` 掩盖了真实退出码——已一并修复）。
- 排查方法论：先用**零并发的交互式 import** 排除"并发/GPFS 争用"假设（一次性、决定性地把假设范围缩小），再用 `find -newer` 判断安装是"进行中"还是"已卡死"。
- **修复**：从 dist-info 目录名直接重建 158 个包的 `name==version` 清单，`pip install --ignore-installed --no-deps` 批量重装，处理了两个坑（setuptools 必须最先修，否则 sdist-only 包装不了；`--ignore-installed` 换版本号时会留下旧版本的 dist-info 残留，需手动清理）。
- **验证到位**：不只是 `import` 成功，而是真实符号访问（`from xgboost import XGBRegressor` 等）+ 在真实 A100 GPU 上跑一次 matmul 确认 CUDA 可用 —— 登录节点/交互节点没有 GPU，纯 CPU smoke test 发现不了 CUDA 层面的问题。
- 期间建立了"轻量隔离 venv"模式（`envs/bde_lite`、`envs/selfies_dg`）作为共享环境不稳定时的兜底方案，避免用共享环境阻塞新实验。

### 1.4 训练/流程规范固化为长期约束
- 明确要求：任何模型训练脚本都必须保留训练曲线、产出完整可视化诊断（parity/residual/训练曲线/误差结构）、导出最差预测样本，而不是只打印一个 MAE。
- 强化既有习惯：一图一文件（不做多面板拼图）、结果文件只加不删/不覆盖、绘图脚本必须登记进索引，空结果（null result）也要记录下来，不能只记正向发现。

## 2. 科学建模成果

### 2.1 g-xTB 全溶剂化中试：确定了正确的几何路线
完成 2192 个分子的 A/B 路线对比（job 24320345 + 补算）：
- **路线 A（funnel_v3 几何 + g-xTB-COSMO 单点）**：MAE **3.26**，R² 0.888，非常接近 DFT，也接近本项目的噪声下限 ~3.2 kcal/mol。
- **GFN2-ALPB**：MAE 13.05（灾难性失败，比取均值预测还差）—— ALPB 溶剂化模型完全不适用这个反应。
- **路线 B（在溶剂中直接用 g-xTB 优化几何）**：收敛率只有 43%，几何会炸裂，**判定不可用**。
- **结论固化为生产规范**：几何一律用 funnel_v3（GFN2/CREST）产出，g-xTB 只做单点 + COSMO，绝不在溶剂中做 g-xTB 几何优化。

### 2.2 Descriptor 搜索：纠正了一次过早的负面结论
- 6-29 前后曾得出"descriptor 搜索已穷尽"的结论（ADCH/QTAIM、morfeus-9、RDKit-434 三个正交描述子族全部落入噪声带）。
- 本周纠正：**不是"这类描述子没用"，而是"无差别整包塞入"这个方法本身有问题**。用 SHAP 提示出的假设（模型已经依赖 P_int/SASA，即弥散/尺寸类信息）去挑选一个**有针对性的 438 维 mordred 子集**（MoRSE/CPSA/极化率/几何指数等），得到真实、超出噪声带的提升：**MAE 1.577 → 1.535**。
- 进一步用 SHAP 重要性 + 相关性剪枝（非 PCA），把 438 维压到 **199 维（合计 271 特征）**，精度几乎不掉（1.525 vs 1.517），**MORDREDSLIM271 成为新的首选生产模型**——同精度、特征数减半。
- 顺带量化了"噪声带"：5 次重新划分种子的 test MAE 为 **1.571 ± 0.013**，5-fold CV 结果一致，为此后所有"这个提升是不是真的"判断提供了统一标尺（<0.02-0.03 视为噪声）。

### 2.3 BDE/BDFE 描述子：本周最大的模型提升
从机理动机出发（Breslow 中间体形成时活化的 C(=O)-H 键）系统性攻关键解离能：
- **GFN2 级别**：raw BDE 有边缘性真实增益（+0.024），但 BDFE（自由能修正版本，理论上更严谨）反而是空结果（+0.007，噪声内）—— 反直觉但被完整验证：ohess 里的低频振动热修正项引入的噪声可能超过其带来的信号。
- **方法不匹配假说**：怀疑 BDE/BDFE 用 GFN2 计算，而模型实际要修正的是 g-xTB 的误差，两者方法不一致稀释了信号。改用 **g-xTB 一致的 BDE/BDFE**（复用项目自有的 hybrid 能量公式，代数复用已缓存值，零新增计算就拿到了母体电子能）。
- **最终结果（2026-07-06 落地）**：
  | 配置 | 特征数 | test MAE | 相对基线 |
  |---|---|---|---|
  | MORDREDSLIM271（基线） | 271 | 1.612 | — |
  | + BDFE(g-xTB) | 273 | 1.605 | -0.007（噪声级，同 GFN2） |
  | + BDE(g-xTB) | 273 | **1.580** | **-0.032（真实，超过 GFN2 版本自己的 +0.024）** |
  | **+ BDE + BDFE(g-xTB)** | 275 | **1.563** | **-0.049（quick-check，约 4 倍噪声带）** |
- 这是本周**唯一超出噪声带 4 倍以上的 quick-check 提升**，随即跑了完整 MLP+3×XGB ensemble（而非 2 员 XGB 快速检验）验证：**真实生产级 test MAE = 1.503**（vs MORDREDSLIM271 的 1.525，Δ-0.022，处于噪声带边缘——比 quick-check 暗示的降幅小，但确认为真实提升）。已提升为新的生产冠军模型。

- **BDE surrogate 模型**（用便宜描述子直接预测 BDE/BDFE，而非每次都跑真实 xtb）已启动训练（job 24436405），用于诊断 BDE 信息是否已被现有特征隐式覆盖，以及未来 screening 新分子时提速。
